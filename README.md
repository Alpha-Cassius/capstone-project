# IIT Patna X Masai AIML (in Hindi) Capstone Project

Welcome to the comprehensive Data Science Capstone Project. This repository is structured into four distinct parts, demonstrating an end-to-end machine learning lifecycle—from raw data processing to the deployment of an LLM-powered explanation feature.

## Repository Structure

The project is divided into four main directories:

### 1. `capstone project - 1` (Data Preprocessing & EDA)
Focuses on cleaning the raw dataset (Titanic survival data) and performing Exploratory Data Analysis. 
- **Key Files**: `src/main.py` (Orchestrates the entire cleaning pipeline)
- **Usage**: 
  ```bash
  cd "capstone project - 1"
  python src/main.py
  ```
- **Output**: Produces the `cleaned_data.csv` used in subsequent parts.

### 2. `capstone project - 2` (Supervised Machine Learning Models)
Focuses on baseline predictive modeling, including Regression (predicting Fare) and Classification (predicting Survival).
- **Key Files**: `src/ml_pipeline.py` (Executes scaling, modeling, threshold tuning, and bootstrapping).
- **Usage**:
  ```bash
  python "capstone project - 2/src/ml_pipeline.py"
  ```
- **Output**: Outputs MSE/R² for regression models, a Confusion Matrix/AUC for classification, and saves the `roc_curve.png`.

### 3. `capstone project - 3` (Advanced Modeling & Ensembles)
Focuses on tree-based models, ensembles (Random Forest, Gradient Boosting), feature ablation, cross-validation, and hyperparameter tuning.
- **Key Files**: `src/advanced_ml.py` (Evaluates ensembles and GridSearch) and `best_model.pkl` (The serialized optimal model).
- **Usage**:
  ```bash
  python "capstone project - 3/src/advanced_ml.py"
  ```
- **Output**: Validates advanced model metrics, produces the optimal `best_model.pkl`, and demonstrates prediction on dummy rows.

### 4. `capstone project - 4` (LLM-Powered Feature)
Focuses on integrating a Generative AI feature (Track C: Model Prediction Explanation Pipeline). It loads the model from Part 3 and asks an LLM to generate structured JSON explanations of the survival predictions while enforcing strict PII guardrails.
- **Key Files**: `src/llm_feature.py`
- **Usage**:
  You must provide a valid LLM API key as an environment variable to run live calls:
  ```bash
  # On Windows (PowerShell)
  $env:LLM_API_KEY="your_api_key_here"
  python "capstone project - 4/src/llm_feature.py"
  ```
  *(Note: If the `LLM_API_KEY` is absent, the script gracefully falls back to a local mock to demonstrate execution and validation logic without crashing).*

## Getting Started

### Prerequisites
Make sure you have Python installed. Install the necessary dependencies by running:
```bash
pip install pandas numpy scikit-learn matplotlib imbalanced-learn requests jsonschema joblib
```
*(Alternatively, run `pip install -r capstone project - 1/requirements.txt` if available)*

### Execution Order
For the complete pipeline experience, it is recommended to run the parts sequentially, as Part 2 relies on data generated by Part 1, and Part 4 relies on the model serialized by Part 3.
